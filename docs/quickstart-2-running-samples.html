Console Application

Building and Running Console Application

  1. If you have not already downloaded them, the samples can be found here.
  2. Open the ConsoleApplicationExample.sln solution file.
  3. Right-click the solution and click Restore NuGet Packages menu item.
  4. Right-click the solution and click Build Solution
  5. Click Debug > Start Debugging to start the app in Debug mode

This starts the console application sample.

To test the sample adapter, set up a Custom Adapter, a Source and a Repository. Follow these steps:

  1. Go to Fusion > Connect screen

  2. Navigate to Sources > Custom Adapters

  3. Click New Custom Adapter

  4. Enter MyCustomAdapter1 into Type and complete the remaining General options as desired

  5. Click Next until you get to Repository Options page

  6. Add Location, UserName and Password options

  7. Click Finish

  8. Create a new Source based on the new custom adapter

  9. Create a new Repository for the new source. In the Location field, define a folder path to scan

    This can be either a local path or a file share. Make sure that the current user has access to this folder.

  10. Optionally, place a breakpoint in MyAdapter, RetrieveFileListAsync and RetrieveFilesDataAsync methods to see the execution.

  11. On the Manage Repositories, click the row for the new repository and then click the update icon that displays in the repository row. This triggers a scan of the folder and processing of files by Fusion.

If you added breakpoints in MyAdapter then the first breakpoint hit should be in RetrieveFileListAsync method. The second - RetrieveFilesDataAsync. Read through the detailed description in the links above.

After the operation finishes, check that all documents were loaded in Fusion.

More details about the Console Application sample can be found here.

Delete files from workspace

To test the delete files from custom adapter workspace, set up a Workspace and a Workbook. Follow these steps:

  1. Go to Fusion > Connect screen

  2. Navigate to Fusion > Manage > Workspaces

  3. Click New Workspace

  4. Enter any name into Name and uncheck Enforce Policy Review option and click on OK to finish setup.

  5. Click on newly created Workspace and go to Content there you will see list of scanned documents.

  6. Select file/s which you want to delete, and click on Manage icon, you will get list of options.

  7. Click on Add selected to Workbook option, it will open window.

  8. It will show list of existing workbook if present else click on NEW WORKBOOK option. In that enter any workbook name in Name field and click on CREATE.

  9. Select newly created workbook from previous opened window and click on ADD button, window will get closed.

  10. After few minuets you can see file count in Item column. and click on blue left arrow highlighted.

  11. It will open side bar in that click Activity and click on Delete icon.

If you added breakpoint in DeleteFilesAsync method of MyAdapter class then the breakpoint gets hit.

After the operation finishes, check that all documents from workbook and from local/network folder gets removed.
